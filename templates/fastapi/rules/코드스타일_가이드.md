# 코드 스타일 가이드 — FastAPI

> FastAPI 프로젝트 공통 컨벤션

---

## 1. 계층 구조 및 책임

| 계층 | 파일 위치 | 역할 |
|------|----------|------|
| **Router** | `api/v1/` | HTTP 엔드포인트, 의존성 주입, 응답 직렬화 |
| **Service** | `services/` | 비즈니스 로직, 트랜잭션 |
| **Model** | `models/` | SQLAlchemy ORM, 상태 변경 메서드 |
| **Schema** | `schemas/` | Pydantic 입출력 검증 |
| **Deps** | `api/deps.py` | `DBSession`, `CurrentUser` 타입 별칭 |

## 2. 네이밍 컨벤션

| 대상 | 규칙 | 예시 |
|------|------|------|
| 모듈 | `snake_case` | `user_service.py` |
| 클래스 | `PascalCase` | `UserService` |
| 함수/변수 | `snake_case` | `get_by_id` |
| 상수 | `SCREAMING_SNAKE_CASE` | `MAX_RETRY` |
| Router 함수 | `{entity}_{action}` | `user_list`, `user_create`, `user_detail` |
| Schema | `{Entity}{Action}` | `UserCreate`, `UserUpdate`, `UserResponse` |
| Service | `{Entity}Service` | `UserService` |

## 3. 패턴 규칙

### Model (SQLAlchemy 2.0)
- `Mapped[T]` + `mapped_column()` 사용 (declarative v2)
- 상태 변경 메서드는 Model 내부에 정의 (DDD)
- `model_config = {"from_attributes": True}` 로 ORM → Schema 변환

### Schema (Pydantic v2)
- `BaseModel` 상속, `Field()` 로 검증
- Create/Update/Response 분리 (Base 상속 활용)
- Update Schema: 모든 필드 `Optional` — `model_dump(exclude_unset=True)` 로 부분 업데이트

### Router
- `APIRouter(prefix="/entity", tags=["태그명"])`
- 의존성 주입: `Annotated[T, Depends(fn)]` 타입 별칭 사용
- 반환: `response_model` 지정, create 시 `status_code=201`

### Service
- `AsyncSession` 기반 async 메서드
- 엔티티 없음 시 `HTTPException(404)` 또는 커스텀 `BusinessError`
- `commit()` → `refresh()` 패턴

### 동적 쿼리
- `select()` 시작 → 조건별 `if` 분기로 `.where()` 추가
- ORM 관계: `selectinload()` / `joinedload()` 로 N+1 방지
