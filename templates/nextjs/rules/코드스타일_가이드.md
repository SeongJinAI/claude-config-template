# 코드 스타일 가이드 — Next.js

> Next.js 프로젝트 공통 컨벤션

## 1. 계층 구조 및 책임

| 계층 | 파일 위치 | 역할 |
|------|----------|------|
| **Page** | `app/` | App Router 페이지, 서버 컴포넌트 (기본) |
| **Component** | `components/` | UI 컴포넌트 (`ui/` 공통, `{feature}/` 기능별) |
| **Store** | `stores/` | Zustand 상태 관리 |
| **API** | `lib/api/` | Axios 클라이언트, API 함수 |
| **Type** | `types/` | TypeScript 인터페이스/타입 |

## 3. 패턴 규칙

### 서버 vs 클라이언트 컴포넌트
- **기본은 서버 컴포넌트** — 데이터 fetching, SEO, 초기 렌더링
- **`'use client'`는 최소한으로** — useState, useEffect, onClick 등 인터랙션 필요 시만
- 클라이언트 컴포넌트는 가능한 **leaf 컴포넌트**로 분리

### 컴포넌트
- Props: `interface {Component}Props` 정의
- `variant`, `size` 등은 union type으로 제한
- `loading`, `disabled` 상태 처리 필수
- react-hook-form + zod 조합으로 폼 검증

### 상태 관리 (Zustand)
- `create<State>()` + `persist` 미들웨어 (인증 등 영속 필요 시)
- 액션 함수는 store 내부에 정의
- 서버 상태는 store가 아닌 서버 컴포넌트에서 직접 fetch

### API 클라이언트
- Axios `create()` 로 기본 설정 (baseURL, headers)
- 요청 인터셉터: 토큰 자동 주입
- 응답 인터셉터: 401 시 토큰 갱신
- API 함수: 도메인별 객체 (`userApi.getList()`)

### 환경 변수
- `NEXT_PUBLIC_` 접두사: 클라이언트 접근 가능
- 서버 전용 변수: 접두사 없이 사용
