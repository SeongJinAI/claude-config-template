# 문서 양식 (Document Format Interface)

> **버전**: v2.0 | **최종 수정**: 2026.03.20

이 디렉토리의 양식은 **인터페이스** 역할을 합니다.
모든 프로젝트는 이 양식의 필수 섹션과 메타데이터 형식을 준수해야 합니다.
프로젝트별 `.claude/rules/문서_템플릿_가이드.md`에서 구체적인 구현을 정의합니다.

## 공통 규칙 (모든 양식 적용)

### 메타데이터 헤더

H1 제목 바로 아래, 인용 부호(`>`) 사용:

```markdown
# {문서 제목}

> **버전**: v{X.Y} | **최종 수정**: YYYY.MM.DD
```

| 규칙 | 형식 |
|------|------|
| 버전 | `v` 접두사 필수 (`v1.0`) |
| 날짜 | `YYYY.MM.DD` (마침표 구분) |
| 스타일 | 인용 부호(`>`) + 볼드 키 |

### 변경 이력

모든 문서의 **마지막 H2 섹션**:

```markdown
## 변경 이력

| 버전 | 일자 | 변경 내용 |
|------|------|----------|
| v1.0 | YYYY.MM.DD | 최초 작성 |
```

- 최신이 상단
- 버전/일자 형식은 메타데이터와 동일

### 섹션 번호

- H2: `## 1. 개요`
- H3: `### 1.1 목적`
- H4: 번호 생략 가능

## 3-Tier 관계

```
거버넌스 (doc-formats/)        ← 인터페이스: 필수 섹션, 메타 형식 정의
    ↓ 구현
프로젝트 (.claude/rules/문서_템플릿_가이드.md)  ← 구현체: 프로젝트별 상세 커스텀
    ↓ 적용
실제 문서 (src/docs/)          ← 인스턴스: 실제 작성된 문서
```

프로젝트가 양식을 커스텀할 때:
- **필수 섹션**: 반드시 포함 (삭제 불가)
- **선택 섹션**: 프로젝트 판단으로 포함/제외
- **섹션 추가**: 프로젝트 필요에 따라 자유롭게 추가
- **메타데이터 확장**: 필수 필드 유지 + 프로젝트별 필드 추가 가능
