# 프론트엔드 프레임워크 선택 가이드

> **버전**: v1.0 | **최종 수정**: 2026.03.23

UI/UX 요구사항에 따라 프론트엔드 프레임워크가 달라집니다.
프로젝트 시작 전 이 가이드를 참고하여 적절한 프레임워크를 선택하세요.

---

## 프레임워크 비교

| | React (Vite) | Next.js |
|---|---|---|
| **렌더링** | CSR (클라이언트) | SSR/SSG/CSR 혼합 |
| **라우팅** | react-router (수동) | App Router (내장) |
| **빌드** | Vite (빠름) | Next.js 번들러 |
| **배포** | 정적 호스팅 (Vercel, S3 등) | Vercel, Node 서버 |
| **SEO** | 기본적으로 불리 | SSR로 유리 |
| **복잡도** | 낮음 | 중간 |
| **거버넌스 템플릿** | `project-templates/react/` | `project-templates/nextjs/` |

## 선택 기준

### React (Vite) 선택

- 백엔드 API가 별도로 존재 (FastAPI, Spring Boot 등)
- SEO가 중요하지 않음 (관리자 페이지, 대시보드, SPA 도구)
- **인터랙티브 UI 중심** (바디 맵, 차트, 드래그앤드롭)
- 빠른 프로토타이핑이 필요
- 프론트엔드 단독 배포 (백엔드와 분리)

### Next.js 선택

- SEO가 중요 (마케팅 페이지, 블로그, 이커머스)
- 서버 컴포넌트가 필요 (데이터 페칭 최적화)
- 풀스택 (API Routes로 백엔드 포함)
- 이미지 최적화, ISR 등 Next.js 고유 기능이 필요

## 이 프로젝트(BodyLogic AI)의 경우

| 요구사항 | 판단 |
|---------|------|
| 인터랙티브 바디 맵 (SVG 터치) | React SPA가 적합 |
| 백엔드가 FastAPI로 분리 | React + API 호출 구조 |
| SEO 필요성 | 낮음 (서비스 도구) |
| 실시간 AI 응답 표시 | CSR이 자연스러움 |

**결론: React (Vite) 선택**
