## 1. Entity 패턴

### 기본 구조

```java
@Entity
@Table(name = "TableName")
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
@AllArgsConstructor
@Builder
public class EntityName {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    // 팩토리 메서드
    public static EntityName from(CreateRequest request) {
        return EntityName.builder()
                .name(request.name())
                .build();
    }

    // Partial Update
    public void update(UpdateRequest request) {
        if (request.name() != null) {
            this.name = request.name();
        }
    }

    // 상태 변경
    public void softDelete() {
        this.deleteYN = "Y";
    }

    // 생명주기 콜백
    @PrePersist
    protected void onCreate() {
        this.createdAt = LocalDateTime.now();
    }

    @PreUpdate
    protected void onUpdate() {
        this.updatedAt = LocalDateTime.now();
    }
}
```

public BigDecimal getTotalAmount() {

private BigDecimal sumOf(BigDecimal... values) {
    BigDecimal sum = BigDecimal.ZERO;
    for (BigDecimal value : values) {
        if (value != null) sum = sum.add(value);
    }
    return sum;
}

## 3. Controller 패턴

### 클래스 어노테이션 순서

`@Tag` → `@RestController` → `@RequestMapping` → `@RequiredArgsConstructor`

### 메서드명 컨벤션

| 작업 | 패턴 |
|------|------|
| 목록 | `{entity}List` |
| 상세 | `{entity}Detail` |
| 등록 | `{entity}Create` |
| 수정 | `{entity}Update` |
| 삭제 | `{entity}Delete` |

### 메서드 형식

```java
@Tag(name = "대분류 - 소분류")
@RestController
@RequestMapping("/domain/entity")
@RequiredArgsConstructor
public class EntityController {

    private final EntityService entityService;

    @Operation(summary = "목록 조회")
    @GetMapping
    public ResponseEntity<ApiResponse<List<ListResponse>>> entityList(
            @AuthenticationPrincipal LoginUser loginUser,
            @Valid @ModelAttribute ListRequest request
    )
    {
        List<ListResponse> response = entityService.getList(request);

        return ResponseEntity.ok(ApiResponse.success(response));
    }
}
```

### CRUD 패턴

```java
@GetMapping                    // 목록
@GetMapping("/{code}")         // 상세
@PostMapping                   // 등록
@PutMapping("/{code}")         // 수정
@DeleteMapping("/{code}")      // 삭제
```

### 응답

```java
return ResponseEntity.ok(ApiResponse.success(data));
return ResponseEntity.ok(ApiResponse.ok("생성 되었습니다."));
```

### Swagger

`@Operation(summary = "...")` 만 사용, `@ApiResponses` 사용 안 함

### N+1 방지 배치 패턴

```java
List<Integer> relatedCodes = entities.stream()
        .map(Entity::getRelatedCode)
        .filter(Objects::nonNull)
        .distinct()
        .toList();

Map<Integer, Related> relatedMap = relatedRepository.findAllById(relatedCodes).stream()
        .collect(Collectors.toMap(Related::getCode, Function.identity()));

return entities.stream()
        .map(entity -> {
            Related related = relatedMap.get(entity.getRelatedCode());
            return ListResponse.from(entity, related);
        })
        .toList();
```

### Service 분리

- `{Domain}Service` — 기존 기능 (조회, 비즈니스 로직)
- `{Domain}ManagementService` — CRUD (등록/수정/삭제)

### 반환 규칙

- create/update/delete는 **void 반환**, Controller에서 `ApiResponse.ok("메시지")`
- JPA dirty checking 활용 — 명시적 `.save()` 호출 생략

## 6. Specification 패턴

### 기본 구조 (null 반환 = 조건 스킵)

```java
public class EntitySpecs {

    public static Specification<Entity> nameContains(String name) {
        if (name == null || name.isBlank()) return null;
        return (root, query, cb) -> cb.like(root.get("name"), "%" + name + "%");
    }

    public static Specification<Entity> statusEquals(Status status) {
        if (status == null) return null;
        return (root, query, cb) -> cb.equal(root.get("status"), status);
    }

    public static Specification<Entity> notDeleted() {
        return (root, query, cb) -> cb.equal(root.get("deleteYN"), "N");
    }
}
```

### Service에서 조합

```java
Specification<Entity> spec = Specification
        .where(EntitySpecs.notDeleted())
        .and(EntitySpecs.nameContains(request.name()))
        .and(EntitySpecs.statusEquals(request.status()));

List<Entity> entities = entityRepository.findAll(spec);
```

### 제네릭 Specs (공통 필드)

여러 엔티티가 동일 필드명을 가질 때 `domain/{name}/specs/` 디렉토리에 제네릭 Specs 정의:

```java
public class CommonSpecs {
    private CommonSpecs() {}

    public static <T> Specification<T> notDeleted() {
        return (root, query, cb) -> cb.equal(root.get("deleteYN"), "N");
    }

    public static <T> Specification<T> statusEquals(String status) {
        if (status == null) return null;
        return (root, query, cb) -> cb.equal(root.get("status"), status);
    }
}
```

## 7. Enum 패턴

```java
// 기본
@Getter
@RequiredArgsConstructor
public enum Status {
    PENDING("대기"), ACTIVE("활성"), DONE("완료");
    private final String description;
}

// Entity에서 사용
@Enumerated(EnumType.STRING)
@Column(name = "Status")
private Status status;

// DB 소문자 저장 시 Converter
@Converter(autoApply = false)
public class StatusConverter implements AttributeConverter<Status, String> {
    @Override
    public String convertToDatabaseColumn(Status attribute) {
        return attribute != null ? attribute.name().toLowerCase() : null;
    }
    @Override
    public Status convertToEntityAttribute(String dbData) {
        return (dbData != null && !dbData.isBlank()) ? Status.valueOf(dbData.toUpperCase()) : null;
    }
}
```

// ErrorCode enum
@Getter

public enum ErrorCode {
    NOT_FOUND("ERR_NOT_FOUND", HttpStatus.NOT_FOUND),

// Service에서 사용
throw new BusinessException(ErrorCode.NOT_FOUND);

## 9. QueryDSL Custom Repository

```java
public interface EntityRepositoryCustom {
    List<ListResponse> findList(ListRequest request);
}

@RequiredArgsConstructor
public class EntityRepositoryCustomImpl implements EntityRepositoryCustom {
    private final JPAQueryFactory queryFactory;

    // 빈 리스트 조건 처리 (버그 방지 필수)
    private BooleanExpression codeIn(StringPath path, List<String> codes) {
        if (codes == null) return null;              // 필터 없음
        if (codes.isEmpty()) return Expressions.FALSE;  // 결과 없음
        return path.in(codes);
    }
}
```
