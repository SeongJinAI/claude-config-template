# 예외 처리 가이드 — FastAPI

> FastAPI 프로젝트의 예외 처리 구현 가이드
> 원칙: `~/.claude/rules/예외처리_원칙.md` 참조

---

## 1. 4계층 → FastAPI 매핑

| 계층 | 예외 | HTTP | 모니터링 |
|------|------|------|---------|
| 요청 검증 | `RequestValidationError` (Pydantic 자동) | 422 | X |
| 인증/인가 | `HTTPException(401)` / `HTTPException(403)` | 401, 403 | X |
| 비즈니스 | `BusinessError(error_code, status_code)` | 정의값 | X |
| DB 정합성 | `IntegrityError` (SQLAlchemy) | 409 | **O** |
| 시스템 | `Exception` (catch-all) | 500 | **O** |

---

## 2. 에러 응답 형식

```json
{"success": false, "message": "ERR_NOT_FOUND_USER", "data": null}
```

---

## 3. 커스텀 예외 구조

- `BusinessError(error_code: str, status_code: int)` — 비즈니스 규칙 위반용
- `app.exception_handler()` 로 전역 등록
- `RequestValidationError` → 422, `BusinessError` → warn 로깅, `IntegrityError` / `Exception` → error 로깅 + 모니터링 전송

---

## 4. 로그 메시지 컨벤션

| 핸들러 | 레벨 | 메시지 |
|--------|------|--------|
| `RequestValidationError` | warning | `요청 데이터 검증 실패: %s` |
| `BusinessError` | warning | `비즈니스 예외: %s` |
| `IntegrityError` | error | `DB 정합성 오류: %s` |
| `Exception` | error | `미처리 예외` (스택트레이스 포함) |
