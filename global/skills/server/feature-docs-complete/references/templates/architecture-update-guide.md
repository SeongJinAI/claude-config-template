# 아키텍처 설명서 업데이트 가이드

구현 완료 후 아키텍처 설명서를 업데이트할 때 다음 항목을 확인합니다.

## 업데이트 체크리스트

1. **메타데이터**: 버전 번호 +1, 최종 수정일 갱신
2. **전체 흐름 (§1)**: 실제 구현된 클래스/메서드명으로 교체. 핵심 아키텍처 결정 테이블 갱신
3. **API 엔드포인트 (§2)**: 기능명세서와 일치하는지
4. **시퀀스 다이어그램 (§3)**: 구현된 호출 순서와 일치하는지
5. **Entity 관계 (§4)**: 실제 연관관계 반영. Entity 테이블 갱신
6. **패키지 구조 (§5)**: 실제 파일 트리와 일치하는지. Policy 클래스 포함
7. **비즈니스 규칙 (§6)**: 기능명세서와 동기화
8. **권한/접근 정책 (§8)**: AgencyAccessPolicy, 도메인 Policy 반영
9. **변경 이력**: 변경 내용 행 추가
