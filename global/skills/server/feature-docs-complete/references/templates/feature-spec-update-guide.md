# 기능명세서 업데이트 가이드

구현 완료 후 기능명세서를 업데이트할 때 다음 항목을 확인합니다.

## 업데이트 체크리스트

1. **메타데이터**: 버전 번호 +1, 최종 수정일 갱신
2. **API 명세 (§2)**: 코드의 실제 엔드포인트와 일치하는지 확인. 권한 컬럼 정확성
3. **데이터 모델 (§3)**: Entity 필드 ↔ DDL ↔ 문서 테이블 동기화. 컬럼명 PascalCase
4. **상세 API 문서 (§4)**: Request/Response 필드가 실제 DTO와 일치하는지
5. **비즈니스 규칙 (§5)**: 구현 중 추가/변경된 규칙 반영. 상태 전이 검증 흐름도
6. **에러 코드 (§6)**: ErrorCode enum 실제 값과 HTTP 상태 일치. ERROR_MESSAGES.md 동시 확인
7. **관련 파일 (§7)**: 실제 파일 경로 정확성. Policy 클래스 포함 여부
8. **변경 이력**: 변경 내용 행 추가 (최신 상단)

## 저장 위치

```
src/docs/specs/{기능명}_기능명세서.md
```

## 파일명 규칙

`{기능명}_기능명세서.md` — 한글, 언더스코어 구분

## 동시 업데이트 대상

- `src/docs/ERROR_MESSAGES.md` — 신규 에러 코드 추가 시
- `src/docs/architecture/{기능명}_아키텍처_설명서.md` — 구조 변경 시
- `src/docs/user-guide/{기능명}_사용자매뉴얼.md` — 사용법 변경 시
