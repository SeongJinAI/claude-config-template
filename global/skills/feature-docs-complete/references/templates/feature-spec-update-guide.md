# 기능명세서 업데이트 가이드

구현 완료 후 기능명세서를 업데이트할 때 다음 항목을 확인합니다.

## 업데이트 체크리스트

1. **API 엔드포인트**: 코드의 실제 엔드포인트와 일치하는지 확인. 추가/변경/삭제된 엔드포인트 반영
2. **요청/응답 DTO**: 실제 DTO 필드와 일치하는지 확인. 타입, 필수 여부, 기본값 반영
3. **검증 규칙**: 코드의 @Valid, 커스텀 검증 로직이 체크리스트에 반영되었는지
4. **에러 코드**: ErrorCode enum의 실제 값과 HTTP 상태 코드 일치 확인
5. **검증 흐름도**: 실제 검증 순서와 일치하는지 확인
6. **비즈니스 규칙**: 구현 중 추가/변경된 규칙 반영
7. **status**: `draft` → `confirmed`로 변경

## 변경 표시 규칙
