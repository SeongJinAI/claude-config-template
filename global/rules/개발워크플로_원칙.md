# 개발 워크플로 원칙

> 모든 프로젝트에 적용되는 개발 순서 및 산출물 원칙

---

## 1. 개발 순서

```
문서 → 코드 → 매뉴얼
```

1. 기능명세서 작성 (API 명세, 검증 흐름도, 검증 체크리스트)
2. 아키텍처 설명서 작성 (전체 흐름, 시퀀스, Entity 관계)
3. 코드 구현
4. 사용자매뉴얼 작성
5. 부수 문서 업데이트 (에러 메시지 문서 등)

---

## 2. 필수 산출물

기능 개발 완료 시 다음 산출물을 반드시 작성한다.

| 산출물 | 설명 |
|--------|------|
| **기능명세서** | API 명세, 검증 흐름도(ASCII), 검증 체크리스트 |
| **아키텍처 설명서** | 전체 흐름(Big Picture), 시퀀스 다이어그램, Entity 관계 |
| **사용자매뉴얼** | 최종 사용자 관점 사용법 |
| **에러 메시지 문서** | 신규 에러 코드 추가 |

### 기능명세서 필수 내용

- API 엔드포인트 목록 (Method, URL, 요청/응답)
- 검증 흐름도 (ASCII 다이어그램)
- 검증 체크리스트 (항목별 조건/에러코드)

### 아키텍처 설명서 필수 내용

- 전체 흐름 (Big Picture)
- 시퀀스 다이어그램 (주요 흐름)
- Entity 관계도
- 테스트 시나리오
- 권한 구분 (해당 시)

---

## 3. 인수인계 (HANDOFF.md)

### 규칙

- `/clear` 명령어 입력 시 **반드시** HANDOFF.md를 업데이트한다
- 날짜/시간(YYYY.MM.DD HH:mm)으로 구분, 최신이 상단

### 필수 기록 항목

1. **완료된 작업** — 이번 세션에서 완료한 작업
2. **다음 작업** — 다음 세션에서 해야 할 작업
3. **주의사항** — 알아야 할 특이사항, 주의점
4. **관련 파일** — 수정/생성된 주요 파일 목록

**IMPORTANT**: `/clear` 명령어 시 HANDOFF.md 업데이트는 **필수**입니다.
