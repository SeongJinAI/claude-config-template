# 예외 처리 원칙

> 모든 프로젝트에 적용되는 프레임워크 무관 예외 처리 원칙

---

## 1. 예외 4계층 분류

모든 예외는 **발생 원인**에 따라 4계층으로 분류한다.

| 계층 | 유형 | HTTP | 원인 |
|------|------|------|------|
| **요청 검증** | 입력값 검증 실패 | 400 | 클라이언트가 잘못된 요청을 보냄 |
| **인증/인가** | 인증 실패, 권한 부족 | 401, 403 | 인증 실패 또는 권한 부족 |
| **비즈니스 로직** | 비즈니스 규칙 위반 | 도메인 정의 | 의도된 흐름 (중복 데이터, 상태 불일치 등) |
| **인프라/시스템** | 코드 버그, 인프라 장애 | 409, 500 | DB 제약 위반, 파일 I/O, 미처리 예외 |

---

## 2. 에러 모니터링 전송 기준

### 핵심 원칙: "예상된 오류는 보내지 않는다"

```
예상된 오류 (모니터링 X) → 사용자 행동의 자연스러운 결과, 프론트가 처리
예상치 못한 오류 (모니터링 O) → 코드 버그 또는 인프라 장애 가능성, 개발자 확인 필요
```

### 판단 기준 3단계

| 순서 | 질문 | Yes | No |
|------|------|-----|-----|
| 1 | 프론트가 이 에러를 처리하도록 설계되었는가? | 전송 X | 다음 |
| 2 | 코드에서 의도적으로 throw하는가? | 전송 X | 다음 |
| 3 | 코드 버그 또는 인프라 문제 가능성이 있는가? | **전송 O** | 전송 X |

### 계층별 기본 매핑

| 계층 | 모니터링 | 이유 |
|------|---------|------|
| 요청 검증 (400) | X | 클라이언트 입력 오류 |
| 인증/인가 (401, 403) | X | 정상 시나리오 |
| 비즈니스 로직 | X | 의도된 비즈니스 흐름 |
| DB 정합성 위반 (409) | **O** | 코드 검증 누락 또는 동시성 이슈 가능성 |
| 시스템 오류 (500) | **O** | 예상치 못한 서버 오류 |

### 모니터링을 보내지 않는 이유 (비즈니스 예외)

1. **노이즈 폭증**: 사용자 잘못된 입력마다 알림 → 진짜 장애가 묻힘
2. **쿼터 낭비**: 유료 모니터링 이벤트 수 제한
3. **의미 없는 알림**: 프론트에서 이미 처리하는 에러

### DB 정합성 위반을 보내는 이유

- DB 제약 위반 = **코드 레벨 검증 누락** 또는 **동시성 이슈** 가능성
- HTTP 409(4xx)이지만 **코드 개선 포인트 추적** 목적으로 예외적 전송

## 3. 로깅 레벨 기준

| HTTP 상태 | 로그 레벨 | 근거 |
|-----------|----------|------|
| 4xx | `warn` | 클라이언트 원인 — 서버는 정상 동작 중 |
| 5xx | `error` | 서버 원인 — 즉시 확인 필요 |
| DB 정합성 위반 (409) | `error` | 4xx이지만 코드 버그 가능성 → error로 격상 |

### 로그 메시지 컨벤션

- **자국어(한글)** 통일 권장
- `"[원인 요약]: {}"` 형식
- 예: `"인증 실패: {}"`, `"비즈니스 예외: {}"`, `"DB 정합성 오류: {}"`

## 4. 에러 코드 컨벤션

### 네이밍

```
ERR_ + SCREAMING_SNAKE_CASE
```

| 유형 | 예시 |
|------|------|
| 데이터 없음 | `ERR_NOT_FOUND_USER` |
| 필수값 누락 | `ERR_REQUIRE_NAME` |
| 권한 없음 | `ERR_UNAUTHORIZED` |
| 중복 데이터 | `ERR_ALREADY_EXISTS` |
| 잘못된 상태 | `ERR_INVALID_STATUS` |

### 중앙 관리 원칙 (Single Source of Truth)

에러 코드는 반드시 **enum 또는 상수 객체**에서 중앙 관리한다. 문자열 리터럴 직접 사용을 금지한다.

```
// Good — 중앙 관리
ErrorCode.NOT_FOUND_USER

// Bad — 문자열 리터럴
"ERR_NOT_FOUND_USER"
```
